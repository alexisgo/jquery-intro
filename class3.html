Review: jQuery Actions

The foundation of all jQuery is combining a selector with an action: 


          $('selector').action();

Review: HTML events

Events occur on a webpage via user interaction

Common Events:

  • mouseenter -- mouse goes inside an element
  • mouseleave -- mouse leaves an element
  • click -- mouse clicks an element
  • keydown -- the user first presses a key on the keyboard.
  • scroll -- the user scrolls
  • Other events

Using jQuery UI

To use jQuery UI, you must include:

  1. A link to the jQuery UI JavaScript library
  2. A link to the default jQuery UI theme, which lives in a CSS file.

Including jQuery UI

  • First, we include a link to the jQuery UI JavaScript library:

AJAX:
Asynchronous JavaScript and XML

  • AJAX = Asynchronous JavaScript and XML
  • Method to communicate to a server or API
  • Asynchronous means:
    • I ask Twitter for all the tweets ever! That will take a while
    • My whole website could be locked up while I wait!
    • Or, my call can be 'asynchronous' and my website will just listen for Twitter's response with one ear, but go about normal business until the response arrives.
  • Requests and results can be in JavaScript or XML

jQuery's load() action

jQuery provides a main ajax() action to perform AJAX requests, but it also has shorthand methods to perform more common kinds of AJAX requests with less code.

One of those methods is load(), and we can use it to inject HTML from one file into another, to save us time. 


  $(document).ready(
    function(){
      $(".sidebar").load('sidebar.html');
     }
  );

jQuery's load() action

Let's say you have a single file of HTML called "sidebar.html" that lays out your sidebar. You want this sidebar present in every single page on your website.

Rather than including the HTML code that makes up this sidebar on every page of the site, we can instead use the jQuery load() action to inject the HTML from an external file (sidebar.html) each time.


    $(document).ready(
      function(){
        $(".sidebar").load('sidebar.html');
       }
    );

The "Same Origin Policy"

Due to browser security restrictions, most AJAX requests are subject to the same origin policy

The load() request can not successfully retrieve data from a different domain, subdomain, or protocol.

What this means in practice is that the HTML file you pass to the load() action must live on the same domain as the file making the call to load().

jQuery's load() action

We can use the jQuery load() action to inject the HTML from an external file (sidebar.html) each time.


      $(document).ready(
        function(){
          $(".sidebar").load('sidebar.html');
         }
      );

The advantage of this approach is that (1) it saves you time and (2) when you need to make changes to the sidebar, you just change that one sidebar.html file!

jQuery's load() action

If we only want to grab one section of the HTML page we are trying to load, we can specify the name of the element in the string we pass to load()

Then, the load() action will grab that element (and everything contained in it) and ignore the rest of the HTML page. 


  $(".sidebar").load('sidebar.html #grabThisElement');
