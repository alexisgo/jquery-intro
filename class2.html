HTML events

Events occur on a webpage via user interaction

Common Events:

  • mouseenter -- mouse goes inside an element
  • mouseleave -- mouse leaves an element
  • click -- mouse clicks an element
  • keydown -- the user first presses a key on the keyboard.
  • scroll -- the user scrolls
  • Other events

Document Ready

One of the most common events we'll want to pay attention to is when the webpage has loaded

Webpages take time to load

Almost always, you don't want the JavaScript to be called until the page is loaded

Document Ready

Document ready is a method called when the page is loaded 

  $(document).ready(
     function() {
     }
  );
Note: The function() inside is an "anonymous function". It has no name, but still performs like a function.

Document Ready

Because we typically want to manipulate our pages with jQuery after the website has finished loading, all of our jQuery code will live inside this Document ready action. 

  $(document).ready(
     function() {
    
  // all our jQuery for the page will live inside here
  
     }
  );

Example: Fading Elements In

  • We can fade in an element as soon as the page loads by combining $(document).ready() with the action fadeIn()
    • NOTE that for fadeIn() to work, the element must originally be hidden by having display:none; set in its CSS.

$(document).ready(
   function(){
     var img = $('#mainpic');
     img.fadeIn();
   }
);

Updating Attributes and Styles with jQuery

One common use of jQuery is to change an HTML element's style following an event.

For example, you may want to swap one image for another in response to a user clicking on it.

We can modify HTML attributes with the action attr().

We can modify CSS styles with the action css().

Updating attributes with jQuery

Assume we have the following HTML: 

 <img id="mainpicture" src="http://girldevelopit.com/assets/pink-logo.png">
Getting and setting attributes via jQuery: 

  var img = $('#mainpicture');
  
  // gets the value of the src attribute
  var src = img.attr('src');

  //sets the value of the src attribute
  img.attr('src', 'newphoto.png');

Swapping Images In Response to a Button Click

As of right now, we can only swap the image once.

It would be nice to add some code that would allow the image to flip back and forth.

To do that, we need a way of knowing which image is currently there, so we know which one to change it to.

We can accomplish this through the attr()'s getter and some if/else statements.

Swapping Images In Response to a Button Click

In the swapImages() function, let's find out what the current value of the src attribute is, by adding this line in orange:

                                            
function swapImages() {
  var img = $('#kitten');

  // get the value of the current src attribute  
  var imgattr = img.attr('src');
}

Adding If/Else

We can use if/else statements to swap images, depending on which image is currently visible.

                                            
  function swapImages() {
    var img = $('#kitten');

    // get the value of the current src attribute  
    var imgattr = img.attr('src');
    
    var kitten1 = "http://placekitten.com/200/200";
    var kitten2 = "http://placekitten.com/50/300";

    if (imgattr === kitten1) {
      img.attr('src', kitten2);
    }
    else {
      img.attr('src', kitten1);
    }
  }

If/Else Statement

You can use else if to have multiple choices


  var age = 20;
  if (age >= 35) {
    console.log('You can vote AND hold any place in government!');
  } else if (age >= 25) {

  } else if (age >= 18) {

  } else {

Running this jQuery code:


 $('.hi').css('border', '5px solid black');

Will update our HTML to have an inline style:


  <div class="hi" style="border: 5px solid black;">hi</div>

This is a bit messy, and violates the principle of separating HTML and CSS. But, there is a better way!

Updating CSS: A Better Way

Rather than directly updating the CSS as we saw on the previous slide, it's better to modify styles on your page by toggling existing CSS classes on and off.
This way, we are not adding CSS via an inline style, which is messy.
We are also preserving the separation of Content (HTML) and formatting (CSS) when we take this approach.

Updating CSS: A Better Way

If we've previously defined a CSS selector to set a size... 

                .shrink {
                 width: 200px;
                }
We can leverage that class, and trigger the change by simply adding the class via jQuery: 

                var img = $('#mainpic');
                img.addClass('shrink');

Combining events

If you want multiple events to happen on the same element, you should use the bind() action


  $('.box').bind( 
    {
        click: function() {
          $(this).css('background-color', 'green')
        },
        mouseenter: function() {
          $(this).css('background-color', 'purple')
        },
        mouseleave: function(){
          $(this).css('background-color', 'orange')
        }
    }
  );

What is an object?

Objects

Objects are a data type that let us store a collection of properties and methods.

The simplest object is just the empty curly braces {}

In more typical objects, the curly braces {} will contain one or more property/value pairs, separated by commas.


  var objectName = { 
    propertyName: propertyValue,
    propertyName: propertyValue
  };

Objects

Objects are a data type that let us store a collection of properties and methods.


    var objectName = { 
      propertyName: propertyValue,
      propertyName: propertyValue,
      propertyName: propertyValue
    };
                      

    var charlie = {
      age: 8,
      name: "Charlie Brown",
      likes: ["baseball", "The little red-haired girl"],
      pet: "Snoopy"
    };

Objects -- returning values

Access values of "properties" using "dot notation":


  var charlie = {
    age: 8,
    name: "Charlie Brown",
    likes: ["baseball", "The little red-haired girl"],
    pet: "Snoopy"
  };
                

  var pet = charlie.pet;

Objects -- returning values

Or using "bracket notation" (like arrays): 

  var name = charlie['name'];
Non-existent properties will return undefined: 

  var gender = charlie.gender

Objects -- changing values

Use dot or bracket notation with the assignment operator to change objects.

Change existing properties: 

                    charlie.name = "Chuck";
Or add new properties: 

                    charlie.gender = "male";
You can also delete properties: 

                    delete charlie.gender;
